# GitNexus CLI

Use the project-local runner when it exists:

```bash
node .gitnexus/run.cjs <command>
```

The runner is generated by GitNexus and chooses an available invocation path
for the current machine. If `.gitnexus/run.cjs` is missing in a fresh clone,
bootstrap it once from the repository root:

```bash
npx gitnexus analyze --skip-agents-md --skip-skills
```

## Commands

### Analyze

Build or refresh the GitNexus index:

```bash
node .gitnexus/run.cjs analyze --skip-agents-md --skip-skills
```

Run after major code changes, after a stale-index warning, or when the MCP
server cannot answer graph questions accurately. Keep the skip flags so
GitNexus refreshes the graph without generating `CLAUDE.md` or `.claude/skills`.

Useful flags:

| Flag | Effect |
| --- | --- |
| `--force` | Force a full re-index. |
| `--embeddings` | Enable embedding generation for semantic search. |
| `--drop-embeddings` | Drop existing embeddings during rebuild. |

### Status

Check index freshness and counts:

```bash
node .gitnexus/run.cjs status
```

### Clean

Delete the local GitNexus index and unregister the repo:

```bash
node .gitnexus/run.cjs clean
```

Use only when the index is corrupt or the user explicitly wants GitNexus
removed/rebuilt. Add `--force` only when non-interactive cleanup is intended.

### Wiki

Generate documentation from the graph:

```bash
node .gitnexus/run.cjs wiki
```

This may require an LLM API key in `~/.gitnexus/config.json`.

### List

List registered indexed repositories:

```bash
node .gitnexus/run.cjs list
```

The Codex MCP tool `list_repos` provides the same repository discovery data.

## Codex Workflow

After indexing, use `list_repos`, read `gitnexus://repo/{name}/context`, then
use the task-specific GitNexus skills for exploring, debugging, impact
analysis, or refactoring.
